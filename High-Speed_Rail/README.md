# High-Speed_Rail

台灣高鐵自動訂票 Chrome 擴充功能。

這個專案會在高鐵訂票網站自動填入查詢條件、票種、乘客資料與聯絡資訊，透過本機 OCR 服務辨識驗證碼，並在完成訂位後發送 Discord 通知。

## 功能

- 自動填入起程站、到達站、日期、最早出發、最晚出發
- 自動設定票種與張數
- 自動帶入乘客身分證字號、手機、Email
- 自動將驗證碼圖片送到本機 OCR 服務
- 自動挑選符合時間範圍的第一班車
- 訂位成功後發送 Discord Webhook 通知

## 專案檔案

- `manifest.json`：Chrome 擴充功能設定
- `index.html`：擴充功能設定介面
- `index.js`：設定資料讀取、儲存與預設值處理
- `content.js`：高鐵網站自動化流程
- `notify.js`：Discord 通知內容與送出邏輯

## 安裝方式

1. 開啟 Chrome 或 Edge。
2. 進入擴充功能管理頁。
3. 開啟「開發人員模式」。
4. 點擊「載入未封裝項目」。
5. 選擇本專案資料夾。

## 使用方式

1. 點開擴充功能 popup。
2. 填入訂票條件與個人資料。
3. 點擊「儲存設定」。
4. 開啟「啟用自動訂票」。
5. 前往 `https://irs.thsrc.com.tw/`。
6. 等待擴充功能自動執行查詢、選車次、填資料與通知。

## 介面欄位說明

| 欄位 | 說明 |
| --- | --- |
| `日期` | 搭乘日期 |
| `最早出發` | 可接受的最早出發時間 |
| `最晚出發` | 可接受的最晚出發時間 |
| `第一步查詢秒數` | 驗證碼已填入後，自動送出查詢的輪詢秒數 |
| `第二步重整秒數` | 查不到符合班次時重新整理頁面的秒數 |
| `起程站` | 出發站 |
| `到達站` | 目的站 |
| `票種` | 票種代碼對應的中文票別 |
| `張數` | 訂票張數 |
| `乘客 1 身分證字號(必填)` | 第一位乘客身分證字號 |
| `乘客 2 身分證字號` | 第二位乘客身分證字號，可留空 |
| `手機(必填)` | 聯絡手機，也會帶入會員手機欄位 |
| `Email（必填）` | 聯絡 Email |
| `Discord Webhook URL（選填）` | 成功通知用的 Discord Webhook |

## 票種代碼對照

| 代碼 | 票種 |
| --- | --- |
| `F` | 全票 |
| `H` | 孩童票 |
| `W` | 愛心票 |
| `E` | 敬老票 |
| `P` | 大學生票 |

高鐵頁面的票種列順序如下：

- `rows:0`：全票
- `rows:1`：孩童票
- `rows:2`：愛心票
- `rows:3`：敬老票
- `rows:4`：大學生票

## 前置需求



### 1. Discord Webhook

如果要接收訂位成功通知，請在 UI 中填入 `Discord Webhook URL（選填）`。

如果這個欄位留空，訂位流程仍可執行，只是不會送出 Discord 通知。

## Discord 通知內容

通知內容目前包含：

- 路線，例如 `台中 → 台北`
- 訂位代號
- 日期與發車 / 到站時間
- 座位資訊
- 票券資訊
- 付款期限

通知顯示名稱為 `高鐵訂位助手`，訊息內容為 `🚄 高鐵購票成功通知！`。

## 自動化流程概要

1. 從 `chrome.storage.sync` 讀取設定。
2. 自動填入高鐵首頁查詢條件。
3. 先把所有票種張數歸零，再設定目標票種與張數。
4. 取得驗證碼圖片並送到本機 OCR。
5. 驗證碼有值後自動送出查詢。
6. 在結果頁篩選符合時間區間的第一班列車。
7. 自動勾選同意、填入身分證、手機、Email 等欄位。
8. 完成訂位後送出 Discord 通知。

## 注意事項

- 這個專案依賴高鐵網站現有 DOM 結構，若官方改版，`content.js` 的 selector 可能需要同步更新。
- `乘客 1 身分證字號`、`手機`、`Email` 目前應視為必填，否則後續流程可能卡住。
- `乘客 2 身分證字號` 只有在雙人以上情境才需要填。
- OCR 服務位址目前寫死為 `http://127.0.0.1:5000/ocr`，若服務位址不同，需要修改程式。
- 如果沒有填寫 Discord Webhook URL，通知功能會自動略過，不會中斷訂票流程。
- 使用自動化工具前，請自行確認是否符合目標網站的使用規範。
