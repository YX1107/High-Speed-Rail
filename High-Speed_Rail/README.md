# High-Speed_Rail

台灣高鐵訂票自動化 Chrome 擴充功能。

這個專案會在高鐵訂票頁面自動填入查詢條件、票種與乘客資料，嘗試透過本機 OCR 服務辨識驗證碼，並在訂票完成後送出 Discord 通知。

## 功能

- 自動填入起訖站、日期、最早與最晚出發時間
- 自動選擇票種與張數
- 自動帶入乘客身分證字號
- 呼叫本機 OCR 服務辨識驗證碼
- 在符合時間範圍的車次中自動選擇第一班
- 訂票成功後送出 Discord Webhook 通知

## 專案結構

- `manifest.json`：Chrome Extension 設定
- `index.html`：擴充功能設定頁面
- `index.js`：設定頁面的資料讀寫邏輯
- `content.js`：高鐵網站自動化流程
- `notify.js`：訂票成功後的 Discord 通知

## 安裝方式

1. 開啟 Chrome 或 Edge 的擴充功能管理頁
2. 啟用「開發人員模式」
3. 選擇「載入未封裝項目」
4. 指向本專案資料夾

## 使用方式

1. 點開擴充功能 popup
2. 設定日期、起訖站、時間區間、票種、張數與乘客資料
3. 開啟自動訂票
4. 前往 `https://irs.thsrc.com.tw/`
5. 擴充功能會在頁面載入後自動執行

## 票種代碼對照

- `F`：全票
- `H`：孩童票 `(6-11)`
- `W`：愛心票
- `E`：敬老票 `(65+)`
- `P`：大學生票

這個對照也反映在高鐵頁面的票種順序：

- `rows:0` → 全票
- `rows:1` → 孩童票
- `rows:2` → 愛心票
- `rows:3` → 敬老票
- `rows:4` → 大學生票

## 設定欄位說明

- `date`：搭乘日期
- `time`：最早出發時間
- `latestTime`：最晚出發時間
- `start`：起程站
- `end`：到達站
- `ticketType`：票種代碼
- `count`：張數
- `p1`：乘客 1 身分證字號
- `p2`：乘客 2 身分證字號
- `step1QuerySeconds`：第一步查詢送出重試秒數
- `step2ReloadSeconds`：第二步無符合車次時的重新整理秒數

## 依賴與前置需求


###  Discord Webhook

`notify.js` 內的 `webhookUrl` 需要替換成你自己的 Discord Webhook URL，否則不會收到通知。

## 注意事項

- 專案目前有部分個人資料欄位寫死在程式內，例如手機與 email，使用前請先確認是否需要改成自己的資料
- 高鐵網站結構若有變動，`content.js` 的 selector 可能需要同步更新
- 使用自動化工具前，請自行確認是否符合高鐵網站的使用規範

## 開發建議

如果你接下來要維護這個專案，建議優先做這幾件事：

- 把手機、email、Webhook URL 改成從設定頁讀取
- 將 OCR 服務位址做成可設定欄位
- 補上更清楚的錯誤訊息與日誌
- 修正目前檔案內出現的編碼亂碼問題
